A full-stack movie platform management system built with **Spring Boot**, **Thymeleaf**, and **Oracle Database**. The application provides complete CRUD management for films, clients, actors, and their interactions — including viewings, votes, comments, and structured feedback — complemented by an analytics dashboard that leverages Oracle stored procedures, functions, triggers, and views to deliver intelligent client profiling and movie recommendations.

---

## ✨ Features

### Core Entity Management
- Full CRUD operations for **Movies** (title, description, category, release date, auto-calculated rating)
- Full CRUD operations for **Actors** (stage name, real name, date of birth)
- Full CRUD operations for **Clients** (contact details, city, email with uniqueness constraint)
- Management of **Movie Versions** (format, resolution, language, subtitles) linked to films
- Management of **Movie Cast / Distribution** (many-to-many: film ↔ actor with role)

### User Interaction Tracking
- **Viewings** — log client viewing sessions per film version, with duration (minutes) and status (`finalizat`, `in_progres`, etc.)
- **Votes / Ratings** — clients rate films on a 1–5 scale; a database trigger automatically recalculates the film's aggregate rating on every vote change
- **Comments** — free-text reviews linked to either a film or an actor (mutually exclusive, enforced by a `CHECK` constraint)
- **Structured Feedback** — clients select from predefined feedback options per film

### Analytics Dashboard
- **Client Profiling** — rich per-client view combining viewings, votes, comments, sentiment analysis, and selected feedback in a single Oracle view (`v_profil_client`)
- **Movie Recommendations** — personalised list of unwatched films in the client's preferred category, generated by a stored procedure (`proc_genereaza_recomandari`) ranked by rating
- **Frequent Actors** — identifies which actors appear most in a client's viewing history, via an Oracle function returning a `SYS_REFCURSOR`
- **Sentiment Analysis** — Oracle PL/SQL function (`fnc_sentiment`) classifies comment text as *pozitiv*, *negativ*, or *neutru* using keyword matching in Romanian and English
- **Business Rule Enforcement** — a `BEFORE INSERT` trigger blocks votes unless the client has a completed viewing for that film (`stare = 'finalizat'`)

### User Experience
- Flash messages for all create / update / delete operations (success and error)
- Graceful Oracle constraint violation handling with user-friendly error messages (ORA-02292, ORA-20011)
- Responsive UI built with Bootstrap 5.3 and Bootstrap Icons
- Card-based navigation dashboard grouping entities by domain

## 🚀 Installation & Setup

### Prerequisites

- **Java 21** (JDK)
- **Apache Maven 3.9+** (or use the included `./mvnw` wrapper — no separate install needed)
- **Oracle Database XE** (Express Edition) running locally on port `1521`, SID `xe`
- An Oracle SQL client (SQL*Plus, SQL Developer, or DBeaver) to execute the setup scripts

### 1. Clone the Repository

```bash
git clone https://github.com/<your-username>/platformaFilme.git
cd platformaFilme
```

### 2. Set Up the Oracle Database

Connect to Oracle as a DBA user (e.g. `system`) and create the application schema user:

```sql
CREATE USER film_user IDENTIFIED BY film_pass;
GRANT CONNECT, RESOURCE, CREATE VIEW TO film_user;
GRANT UNLIMITED TABLESPACE TO film_user;
```

Then connect as `film_user` and execute the scripts **in order**:

```sql
-- 1. Drop existing objects (safe to skip on first run)
@src/main/resources/static/01_drop.sql

-- 2. Create tables
@src/main/resources/static/02_tabele.sql

-- 3. Add unique constraints
@src/main/resources/static/03_constrangeri.sql

-- 4. Create sequences
@src/main/resources/static/04_secvente.sql

-- 5. Create PL/SQL functions (sentiment analysis, category preference, frequent actors)
@src/main/resources/static/05_functii.sql

-- 6. Create triggers (rating auto-update, vote validation)
@src/main/resources/static/06_triggere.sql

-- 7. Create stored procedures (recommendation engine)
@src/main/resources/static/07_proceduri.sql

-- 8. Create analytics view
@src/main/resources/static/08_statistici.sql

-- 9. Create performance indexes
@src/main/resources/static/12_indecsi.sql

-- 10. (Optional) Populate with random seed data
@src/main/resources/static/11_populare_random.sql
```

### 3. Configure the Application

Open `src/main/resources/application.properties` and verify or adjust the values:

```properties
spring.datasource.url=jdbc:oracle:thin:@localhost:1521:xe
spring.datasource.username=film_user
spring.datasource.password=film_pass
spring.datasource.driver-class-name=oracle.jdbc.OracleDriver
server.port=9090
```

### 4. Build & Run

```bash
# Using the Maven wrapper (no separate Maven installation required)
./mvnw spring-boot:run

# Or build a JAR and run it
./mvnw clean package
java -jar target/platformaFilme-0.0.1-SNAPSHOT.jar
```

The application starts on **http://localhost:9090**.

| `spring.datasource.username` | `film_user` | Database user |

```bash
  --spring.datasource.password=<secret> \
  --server.port=8080
```

## 🗄 Database

### Oracle Integration

The application connects to Oracle XE using the official **OJDBC 11** driver. All data access is performed through **Spring's `JdbcTemplate`** with hand-written SQL, giving direct access to Oracle-specific PL/SQL constructs.

### Schema Overview

| Table | Purpose |
|---|---|
| `filme` | Core movie catalogue (title, category, release date, computed rating) |
| `versiuni_film` | One-to-many movie versions (format, resolution, language, subtitles) |
| `clienti` | Platform subscribers (contact details, unique email) |
| `actori` | Actor registry (stage name, real name, birth date) |
| `distributie_film` | Movie cast — composite PK `(id_film, id_actor)` with role |
| `vizualizari` | Viewing sessions — links client → film version, includes duration and status |
| `voturi` | Client ratings (1–5); one vote per client per film (unique constraint) |
| `comentarii` | Text reviews linked to a film **or** actor (XOR constraint) |
| `optiuni_feedback` | Configurable feedback option catalogue |
| `feedback_selectat` | Client selections from feedback options per film |

### Sequences

Eight Oracle sequences handle surrogate key generation (e.g. `seq_film`, `seq_client`, `seq_actor`, `seq_vizualizare`). Each repository calls `nextval` on insert.

### PL/SQL Objects

| Object | Type | Description |
|---|---|---|
| `fnc_sentiment` | Function | Keyword-based sentiment classification (Romanian + English) → `'pozitiv'` / `'negativ'` / `'neutru'` |
| `fnc_categorie_preferata` | Function | Computes a client's preferred genre using a weighted affinity score (views + vote delta + sentiment bonus) |
| `fnc_actori_frecventi` | Function | Returns a `SYS_REFCURSOR` of actors ranked by how often a client has watched films featuring them |
| `proc_genereaza_recomandari` | Procedure | Recommends top-N unseen films in the client's preferred category, sorted by rating |
| `trg_actualizeaza_rating` | Trigger | `AFTER INSERT/UPDATE/DELETE ON voturi` — recalculates average rating on the `filme` table |
| `trg_verifica_vot` | Trigger | `BEFORE INSERT ON voturi` — raises `ORA-20011` unless the client has a `finalizat` viewing |
| `v_profil_client` | View | Comprehensive per-client activity view joining all interaction tables with inline sentiment evaluation |

### Indexes

Eight targeted indexes covering high-cardinality join/filter columns: `vizualizari(id_client)`, `vizualizari(stare)`, `comentarii(id_film)`, `versiuni_film(id_film)`, `filme(categorie)`, `actori(nume_scena)`, and others.

## 🔗 Application Routes

The application follows a consistent MVC URL pattern across all managed entities.

### Navigation

| Route | Method | Description |
|---|---|---|
| `/` | GET | Main dashboard — card navigation to all modules |
| `/statistici` | GET | Analytics dashboard — entry point for all analytics features |

### Movies (`/filme`)

| Route | Method | Description |
|---|---|---|
| `/filme` | GET | List all movies |
| `/filme/nou` | GET | New movie form |
| `/filme/nou` | POST | Create movie |
| `/filme/{id}/editare` | GET | Edit form for movie `{id}` |
| `/filme/{id}/editare` | POST | Update movie `{id}` |
| `/filme/{id}/stergere` | POST | Delete movie `{id}` |

### Actors (`/actori`)

| Route | Method | Description |
|---|---|---|
| `/actori` | GET | List all actors |
| `/actori/nou` | GET / POST | Create actor |
| `/actori/{id}/editare` | GET / POST | Edit / update actor |
| `/actori/{id}/stergere` | POST | Delete actor |

### Clients (`/clienti`)

| Route | Method | Description |
|---|---|---|
| `/clienti` | GET | List all clients |
| `/clienti/nou` | GET / POST | Create client |
| `/clienti/{id}/editare` | GET / POST | Edit / update client |
| `/clienti/{id}/stergere` | POST | Delete client |

### Movie Versions (`/versiuni`)

| Route | Method | Description |
|---|---|---|
| `/versiuni` | GET | List all versions |
| `/versiuni/nou` | GET / POST | Create version (select parent film) |
| `/versiuni/{id}/editare` | GET / POST | Edit / update version |
| `/versiuni/{id}/stergere` | POST | Delete version |

### Cast / Distribution (`/distributie`)

| Route | Method | Description |
|---|---|---|
| `/distributie` | GET | List all cast entries |
| `/distributie/nou` | GET / POST | Assign actor to film with role |
| `/distributie/{idFilm}/{idActor}/editare` | GET / POST | Update role for a film/actor pair |
| `/distributie/{idFilm}/{idActor}/stergere` | POST | Remove actor from film cast |

### Viewings (`/vizualizari`)

| Route | Method | Description |
|---|---|---|
| `/vizualizari` | GET | List all viewing records |
| `/vizualizari/nou` | GET / POST | Log a new viewing session |
| `/vizualizari/{id}/editare` | GET / POST | Edit viewing record |
| `/vizualizari/{id}/stergere` | POST | Delete viewing record |

### Votes (`/voturi`)

| Route | Method | Description |
|---|---|---|
| `/voturi` | GET | List all votes |
| `/voturi/nou` | GET / POST | Submit a vote (triggers DB validation) |
| `/voturi/{id}/editare` | GET / POST | Edit vote |
| `/voturi/{id}/stergere` | POST | Delete vote |

### Comments (`/comentarii`)

| Route | Method | Description |
|---|---|---|
| `/comentarii` | GET | List all comments |
| `/comentarii/nou` | GET / POST | Add a comment (film or actor) |
| `/comentarii/{id}/editare` | GET / POST | Edit comment |
| `/comentarii/{id}/stergere` | POST | Delete comment |

### Feedback Options (`/optiuni-feedback`) & Selected Feedback (`/feedback-selectat`)

Both follow the same GET list / GET+POST form / POST delete pattern as the entities above.

### Analytics (`/statistici`)

| Route | Method | Parameters | Description |
|---|---|---|---|
| `/statistici` | GET | — | Analytics dashboard with client selectors |
| `/statistici/profil` | GET | `idClient` | Full activity profile for a client via `v_profil_client` |
| `/statistici/recomandari` | GET | `idClient`, `numar` (default 5) | Personalised movie recommendations |
| `/statistici/actori-frecventi` | GET | `idClient` | Actors ranked by viewing frequency for a client |

#### Example: Generate Recommendations

```
GET /statistici/recomandari?idClient=3&numar=10
```

Returns up to 10 unwatched films in the client's preferred genre, ranked by rating. Internally calls `proc_genereaza_recomandari` which itself calls `fnc_categorie_preferata`.

#### Example: Client Profile

```
GET /statistici/profil?idClient=7
```

Queries `v_profil_client` (a view that joins viewings, votes, comments, feedback, and runs `fnc_sentiment` inline) and renders a detailed per-film activity table with sentiment labels.

